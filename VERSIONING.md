# VisePanda Versioning

## Current Versions

- Previous baseline: `v0.1.1`
- Minor-series reset: `v0.2.1` (operator directive — all subsequent iterations use `0.2.x`)
- Current iteration: `v0.3.16` (native Explore board connected to live Next.js /api/explore/amap route with dynamic fallback and CanvasPatch candidate pipeline integration)
- Production domain: `go2china.space`

## Rule

- Default product iteration format is `0.3.x` (reset from `0.2.x` at operator request as project main development branch officially shifts to native Android/iOS; Web app series stops at `v0.2.17`).
- Increment the `0.3.x` patch by 1 each iteration unless the operator specifies a different version.
- Every iteration must update `package.json` and `CHANGELOG.md`.
- Use a custom version only when the user explicitly provides one.

The actual, previously-undiagnosed bug: `di/AppModule.kt`'s `OkHttpClient` used bare defaults (10s connect/read/write), but a timed direct test of the production backend took 14.4s and the same request replayed through the app (captured via a new debug-only `HttpLoggingInterceptor`) took 20.2s — meaning every real chat request in this project's history had been silently timing out and falling back to `NativeButlerFallback`'s offline copy, which is why Chat had never once been manually verified against a live AI response despite being "wired up" since v0.3.6. Fixed by raising timeouts to 15s/45s/15s (connect/read/write). Also closed the one real contract gap: added `InlineToolCard`/`InlineToolCardTone` to `ButlerModels.kt` and a `toolCards` field on `AssistantResponse`, with `ButlerScreen.kt` rendering them as a nested card in the chat bubble (`href` deep-linking intentionally not wired since Tools is still a placeholder).
