## 第一部分:宏观分析 —— 定位到体验的映射

### 1.1 定位复述(锚点)

> VisePanda 坚决锁定**非跟团独立自由行(FIT)外国来华游客**,是集行程规划、
> 酒店预订、景点/餐饮选择、实时翻译于一体,深度打通交通、支付、地图等
> 插件能力的**一站式智能旅游管家 AI**。

宏观推论(全部 UI/交互决策的判据):

1. **管家 ≠ 工具箱**。六个平级 tab 是工具箱的形态;管家形态 = 一条对话主线
   (Chat)+ 一个可信的行程实体(Canvas),其余能力在对话中被"递上来"。
2. **一站式 = 不离开**。每次跳出(切 tab、开新页、复制去别的 App)都是
   定位的失败。衡量指标:完成一次决策所需的界面切换次数。
3. **FIT 用户买的是确定性**。签证/支付/联网/语言/行程五大焦虑,每一个都
   必须在 Canvas 一步可达范围内有可见的解决入口(v0.1.55 蓝图的
   "五焦虑一步原则")。
4. **AI 的智能必须可见**。多模型编排、真实 POI、偏好记忆已经在跑,但 UI
   上几乎不可感——智能不可见等于没有。

### 1.2 当前体验与定位的差距(宏观审计)

| # | 差距 | 现状 | 定位要求 | 严重度 |
|---|---|---|---|---|
| G1 | 回答是文字墙 | 结构化回复(headline/highlights/watchOut/nextStep)已有数据,但渲染是普通段落+列表,无视觉分层 | 管家的回答应"一眼可扫":标题大、要点打勾、警示醒目、下一步是按钮 | 高 |
| G2 | Canvas 是被动展示 | Day 卡只能看和开抽屉;修改必须打字 | 管家形态:每张卡自带快捷动作,点一下就让 AI 干活 | 高 |
| G3 | 变化不可感 | AI 更新画布后,除一行 status 文本外无任何视觉信号 | 每次 patch 应有"哪里变了"的动画/高亮 | 高 |
| G4 | 无加载状态 | 等待回复时界面静止,配合此前的慢回复体感极差 | 骨架屏 + 乐观 UI + 打字指示 | 高 |
| G5 | 行程完成度不可知 | 用户不知道"这个行程还缺什么才能出发" | 完成度进度条 + 缺口清单(prep blockers) | 高 |
| G6 | 工具与对话割裂 | 问签证 → 要么 LLM 慢答,要么让用户自己去 Tools | 事实类问题秒出内联工具卡 | 中 |
| G7 | 样式手写、无系统 | 每页手写 CSS,按钮/卡片/胶囊各处形态不一 | 设计 token + 组件库,一处定义处处复用 | 中 |
| G8 | 空/错状态未设计 | 空画布、无结果、请求失败都是简陋文本 | 有插画/引导动作的设计态 | 中 |
| G9 | 移动端为"能用"非"顺手" | v0.1.44 修了布局,但触达/动效/手势未打磨 | 移动端是在华场景主力,需专项打磨 | 中 |
| G10 | 五焦虑入口分散 | 支付/签证/联网提示散落在 alerts 与 Tools | Canvas 上有统一的"出发准备"区 | 中 |

### 1.3 宏观结论:后三轮的主题

以上差距聚成三类,恰好对应三轮迭代:

- **让行程可操作**(G2/G3/G5/G10)→ 第一轮:Canvas 行动层 + 画布 UI 升级
- **让对话像管家**(G1/G4/G6)→ 第二轮:对话体验重塑 + 内联工具卡
- **让界面成体系**(G7/G8/G9 + Tools 功能化)→ 第三轮:设计系统 + 工具组件化

顺序理由:先把"行程实体"做实(用户价值最直接、零外部依赖),再把"对话
主线"做顺(依赖第一轮的完成度数据),最后用设计系统收口(前两轮新增的
组件正好成为组件库的第一批客户,避免先建空系统)。

## 第二部分:微观分析 —— 逐界面 UI 优化清单

> 每条标注目标轮次沿用原始 R1/R2/R3。版本号已顺延: R1=v0.2.6,R2=v0.2.7,R3=v0.2.8。
> 视觉基调不变:Warm New Chinese(纸色/墨色/朱砂/金/苔绿),禁玻璃拟态。

### 2.1 Live Trip Canvas(核心实体)

- **完成度进度条**(R1):`TripSummary` 顶部,0–100%,分段着色
  (<40% 墨灰 / 40–79% 金 / ≥80% 苔绿),旁挂缺口 chips
  ("缺酒店区域""签证未核对"),点 chip = 发送预制意图。
- **Day 卡快捷动作**(R1):卡片底部一行小按钮
  `减负 / 加美食 / 换上午 / 加休息`,走既有 `handleSend` 管道;busy 时禁用。
- **Patch 可视化**(R1):新增卡淡入上滑 240ms;`status:"revised"` 卡左缘
  金色 2px 脉冲一次;`lastUpdatedReason` 以浅金底 toast 短暂浮现。
- **出发准备区**(R1):Canvas 底部固定小节"Before you fly",聚合五焦虑
  相关 alerts + 完成度缺口,每项可勾选"已完成"(勾选回写完成度)。
- **Day 卡信息密度**(R3):时段图标(晨/午/晚)、城市色点、pace 胶囊
  统一为组件库的 `Pill`/`StatDot`。

### 2.2 Chat 对话区(管家主线)

- **结构化回复分块渲染**(R2):headline 用 serif 加粗大一号;highlights
  前置 ✓(苔绿);watchOut 琥珀底浅条 + ⚠;nextStep 独立成朱砂主按钮
  (已有,统一进组件 `MessageBlock`)。
- **等待态**(R2):发送后立即乐观渲染用户气泡;助手位放三点打字指示 +
  Canvas 侧对应区域骨架屏;>8s 显示"正在比对多家模型…"安抚文案。
- **内联工具卡**(R2):`ask_factual` 秒回静态卡(签证/支付/eSIM/汇率/
  地铁/应急),卡上带「加入提醒」「标记完成」,回写 R1 的完成度。
- **模型标签弱化**(R2):现 status 行含模型名,改为小号灰字缀于回复块
  右下角,主 status 行只讲人话("已为你减轻第 2 天强度")。
- **建议 chips**(R2):固定 2 个,来自 `nextStep`/`suggestions`,样式统一
  为 ghost Pill。

### 2.3 Tools(从清单到工具)

- **交互组件优先**(R3):抽屉内 widget 在上、静态清单收进"参考清单"
  折叠区;`ToolCategory` 增可选 `interactive` 描述符,无数据时整体降级为
  纯清单(fallback 保留)。
- **首批三件套**(R3):汇率换算器(已有实时汇率路由)、签证资格问答器
  (护照国→免签/过境/需签证 决策树,纯静态规则)、支付设置向导
  (卡组织选择→分步引导+费率警示)。
- **应急一键化**(R3 若余量):110/120/119 `tel:` 大按钮 + 常用求助短语
  TTS。

### 2.4 移动端专项(贯穿三轮,每轮附带)

- R1:Day 卡快捷动作触达 ≥44px;完成度条在窄屏折行为"环形小表盘+chips"。
- R2:Chat 在移动端为底部停靠 composer,展开成 70dvh sheet;内联卡全宽。
- R3:Tools widget 在 sheet 内可用;安全区与固定底导航复检(沿用 v0.1.44
  级联覆盖模式)。

### 2.5 空/错/边界状态(R2 起逐步)

- 空画布:三个原型入口卡(已有)+ 一行"或直接告诉我你的想法"。
- 模型全部失败:status 行明确"已用离线草稿引擎生成,联网模型恢复后会更好",
  不再只写 mock 字样。
- 无 POI 结果 / 网络失败:统一 `EmptyState` 组件(插画位 + 主动作)。

## 第三部分:前端设计系统迭代规划(R3 落地,R1/R2 预埋)

### 3.1 Token 化(R3)

在 `app/globals.css` 顶部整理为显式 token 层(值即现行视觉,不改观感):

```
--paper/--paper-soft/--ink/--cinnabar/--gold/--sage(已有,补语义别名)
--ok=sage --warn=#B8862E --danger=#9D2F24 --info=#4A6080
间距 4/8/12/16/24/32;圆角 8/12/999;动效 120ms(micro)/240ms(sheet)
全部动效尊重 prefers-reduced-motion
```

### 3.2 组件库首批(R1/R2 先以局部组件出现,R3 归一)

`Button(primary/ghost/quiet)`、`Pill`、`Card`、`Sheet`、`Toast`、
`ProgressMeter`(R1 完成度)、`MessageBlock`(R2 回复分块)、
`ToolCard`(R2 内联卡)、`EmptyState`、`RatingStars`/`PriceLevel`(Explore
已有雏形,收编)。规则:R3 起新界面禁止手写同类样式,旧页面逐步迁移。

### 3.3 动效与反馈准则

- 只做"有含义的动效":出现(240ms)、变更脉冲(1 次)、完成打勾;
  禁装饰性循环动画。
- 每个用户动作 100ms 内必须有可见反馈(按钮态/乐观气泡/骨架)。

## 第四部分:后三轮迭代规划(执行承诺)

> 版本按 `0.2.x` 递增;因本规划轮占用 v0.2.3,原口头排期顺延一号。
> 每轮均:更新全部 7 文档 + 测试/构建通过 + 双分支推送 + fetch 防并行冲突。

### 🔵 R1 = v0.2.6 —— Canvas 行动层 + 画布 UI 升级

**主题:让行程从"能看"变成"能指挥"。**

| 类别 | 交付物 |
|---|---|
| 功能 | `lib/trips/completeness.ts` 完成度评分(路线/酒店/餐食/交通/支付/签证六维);Day 卡快捷动作(减负/加美食/换上午/加休息 → 预制意图走 AI 管道);"出发准备"聚合区 + 可勾选缺口 |
| UI | 完成度 `ProgressMeter`(分段配色);patch 动画(新卡淡入、revised 金色脉冲);缺口 chips;快捷按钮触达 ≥44px |
| 测试 | `completeness.test.ts` 评分断言;快捷动作触发正确意图;动画类名快照 |
| 边界 | 快捷动作绝不直改 canvas;评分纯函数;移动端折行方案 |

**验收**:空行程→原型开局→完成度可见增长;点"减负"画布 3 秒内可见变化且变化处有高亮。

### 🟠 R2 = v0.2.7 —— 对话体验重塑 + 内联工具卡

**主题:让对话像管家,不像终端。**

| 类别 | 交付物 |
|---|---|
| 功能 | `ask_factual` 快通道:签证/支付/eSIM/汇率/地铁/应急直出静态内联卡(跳过 LLM,<150ms);卡上「加入提醒/标记完成」回写完成度 |
| UI | `MessageBlock` 分块渲染(serif headline/✓highlights/⚠watchOut/朱砂 nextStep);乐观气泡 + 打字指示 + Canvas 骨架屏;>8s 安抚文案;模型标签降为小灰字;`EmptyState` 组件 |
| 测试 | 事实类意图绕过 LLM 且 <150ms 渲染卡;分块渲染快照;慢响应安抚文案出现 |
| 边界 | 内联卡数据全部来自既有 `lib/tools` 静态层;LLM 通道行为不回归 |

**验收**:问"Do I need a visa?"秒出卡片并可一键加入行程提醒;任何发送后界面 100ms 内有反馈。

### 🟢 R3 = v0.2.8 —— 设计系统收口 + Tools 交互组件

**主题:让界面成体系,让工具真的能用。**

| 类别 | 交付物 |
|---|---|
| 功能 | Tools 三件套 widget:汇率换算器(实时 API,离线快照)、签证资格问答器(静态决策树)、支付设置向导(分步 + 费率警示);`ToolCategory.interactive` 可选描述符,降级保留静态清单 |
| UI | token 层整理 + 组件库归一(Button/Pill/Card/Sheet/Toast/ProgressMeter/MessageBlock/ToolCard/EmptyState);R1/R2 新组件收编;移动端 sheet 内可用性复检 |
| 测试 | 换算器换算断言;签证决策树多国用例;widget 缺数据降级为清单 |
| 边界 | 不动 provider 抽象;所有 key 服务端;静态清单永不删除 |

**验收**:三个工具在桌面与移动端都能完成真实任务;新组件在 ≥3 个页面复用且无样式漂移。

### 三轮之后的队列(不承诺顺序,视审批/决策)

大众点评/美团接入(等审批)→ 地图 widget(等 `NEXT_PUBLIC_AMAP_MAPS_KEY`)
