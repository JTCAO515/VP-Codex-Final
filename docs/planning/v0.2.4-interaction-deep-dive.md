> 后续三轮代码迭代(因 v0.2.5 融合轮占号,最终顺延为 v0.2.6 / v0.2.7 / v0.2.8)以本文档为

## 第〇部分:交互设计哲学(判据,先于一切规格)

1. **对话是手,画布是脸。** 用户对话时视线在画布上——所以 AI 的每次改动
   必须在画布上"演"出来,而不是在聊天里"说"出来。
2. **可点优于可打。** 凡是能预测的下一步(减负/换一个/加美食/撤销),都
   做成一次点击;打字只留给无法预测的意图。
3. **100ms 反馈定律。** 任何点击/发送,100ms 内必须有可见反应(按钮态、
   乐观气泡、骨架屏);300ms 内没有内容就必须有等待叙事。
4. **一屏一个朱砂。** 朱砂色 = "现在最该点的那个东西",每屏最多一个;
   其余动作用 ghost/quiet 态,保证视线有唯一落点。
5. **动效只为"发生了什么"服务。** 出现、变更、完成三类之外不做动画。

## 第一部分:Chat ↔ Canvas 联动的可见性设计(本轮最重要的交互命题)

### 1.1 变更摘要卡(Change Digest)——联动的核心载体

每次 canvas patch 应用后,助手回复块底部追加一张微型"本次改动"卡:

```
┌─ 本次改动 ────────────────────────────┐
│ ✚ Day 3 新增 · 成都大熊猫基地(上午)   │   ← 每行一个 diff 条目
│ ⟳ Day 2 调整 · 下午行程减为 2 项       │   ← 图标区分 新增/调整/删除
│ ⚑ 新提醒 · 提前 7 天预约故宫           │
└──────────────────────[全部显示在画布→]┘
```

- 数据来源:`applyCanvasPatch` 前后 `TripState` 的浅 diff(day 级 + alert 级),
  纯函数 `lib/canvas/diffTripState.ts`,可单测。
- **点击任一条目 → 画布滚动到对应 Day 卡并触发一次金色脉冲**。这一下点击
  就是"联动"的体感来源。
- 无实质变化时(纯问答)不渲染此卡。

### 1.2 双向悬停联动(桌面)

- 悬停变更摘要卡中的"Day 2"条目 → 画布 Day 2 卡边框亮起(hairline → 金色)。
- 悬停画布 Day 卡 → 聊天里最近一次提及该天的回复块左缘出现 2px 墨色标记。
- 实现:共享 `highlightDayNumber` 状态(workspace 级 useState),CSS 类切换,
  无新依赖。移动端跳过此项(无 hover)。

### 1.3 Patch 后的画布"演出"顺序(staged reveal)

一次 create_trip 的画布呈现不是瞬间全出,而是 600ms 内的编排:

```
t=0      摘要卡(标题/天数/目的地)先淡入
t=120ms  Day 1 卡淡入上滑(8px)
t=160ms  Day 2 卡……(每卡 stagger 40ms)
t=末尾   完成度进度条从旧值动画到新值(300ms ease-out)
```

- adjust_trip 时只有 `status:"revised"` 的卡做金色脉冲(600ms,1 次),
  其余卡不动;画布自动平滑滚动到**第一张**变更卡(`scrollIntoView`,
  `behavior:"smooth"`, `block:"nearest"`)。
- `prefers-reduced-motion`:全部退化为即时渲染 + 静态金色左缘(2s 后消失)。

### 1.4 撤销(undo)——信任的安全网

- 变更摘要卡右上角挂"撤销"quiet 按钮,点击发送预制意图
  `Undo the last change and restore the previous itinerary.`,走既有 AI 管道。
- 前端同时在 workspace 保留上一份 `TripState` 快照;若 AI 通道失败,本地
  快照直接回滚(fallback 不破坏"一切走管道"原则的例外条款,需在 AGENTS
  注明:仅 undo 允许本地快照回滚)。

## 第二部分:Chat 输入与回复的交互规格

### 2.1 输入区(composer)

| 项 | 规格 |
|---|---|
| 提交 | `Enter` 发送;`Shift+Enter` 换行(桌面)。移动端回车即换行,发送靠按钮 |
| 高度 | 初始 1 行(44px),随内容自动长高,上限 5 行后内部滚动 |
| 发送后 | 立即清空、焦点留在输入框、聊天滚到底 |
| busy 态 | 输入框可继续打字,发送按钮转为 spinner;禁止重复提交 |
| 占位文案 | 轮换 3 条场景化提示("Try: 5 days in Chengdu with pandas…"),8s 一换,聚焦即停 |

### 2.2 回复块(MessageBlock)分层规格

```
VisePanda                                   ← 12px 辅助灰
┌──────────────────────────────────────┐
│ 两天成都,熊猫与火锅都排好了            │  ← headline: serif 18px/600, ink
│ 第一天以熊猫基地开场…(正文一段)       │  ← body: 14px/1.55, ink 88%
│ ✓ 熊猫基地 9 点开门,建议 8:40 到      │  ← highlights: 13px, ✓ 苔绿
│ ✓ 火锅店人均 ¥120,已按预算选         │
│ ⚠ 周一博物馆闭馆,已避开               │  ← watchOut: 琥珀底 8% 圆角条
│ ┌ 本次改动(见 1.1)┐                  │
│ [ 要不要加一晚锦里夜市? ]              │  ← nextStep: 朱砂主按钮(本屏唯一)
└──────────────────────────────────────┘
      模型标签 · Zhipu GLM-5              ← 11px 灰 60%,右对齐
```

- 分阶段呈现(伪流式,不改 API):回复到达后 headline 先渲染,body 与
  highlights 以 60ms stagger 逐条淡入——"管家在陈述"的体感,总时长 <400ms。
- 回复中的**实体 chip**(v0.2.7 实现):正文中匹配画布内 POI 名称的词组
  渲染为下划虚线 chip,点击 = 画布定位 + 脉冲(复用 1.2 的高亮通道)。

### 2.3 等待叙事(替代死寂)

| 时间 | 呈现 |
|---|---|
| 0–100ms | 用户气泡乐观上屏;发送按钮 spinner |
| 100ms | 助手位出现三点打字指示;画布侧完成度条进入 shimmer |
| >3s | 打字指示下加一行 12px 灰字:"正在核对真实景点与营业时间…" |
| >8s | 换文案:"多家模型比对中,再等几秒…" |
| 失败 | 打字指示原位变为回复块(fallback 内容),status 行写明"已用离线草稿引擎,联网模型恢复后会更准" |

### 2.4 建议 chips

- 固定 2 枚,ghost Pill(hairline 边、纸底),位于回复块下方而非顶部——
  紧跟语境;点击即发送。
- 首屏空状态例外:3 张原型开局卡(既有)+ 1 行"或直接告诉我你的想法"。

## 第三部分:Canvas 交互规格

### 3.1 Day 卡结构与状态

```
┌ Day 2 · 成都 ────────────── [Balanced] ┐   ← 主行:serif 15px;pace 角标 Pill
│ ● 上午  熊猫基地                        │   ← 时间轴竖线连接三段,12px 圆点
│ ● 下午  人民公园茶馆                    │
│ ● 傍晚  锦里夜市                        │
│ 🍜 火锅 · 🏨 春熙路 · 🚇 地铁          │   ← 次要行:13px 灰 72%,图标 16px
│ [减负] [加美食] [换上午] [+休息]        │   ← 快捷动作:quiet 按钮,高 32px
└────────────────────────────────────────┘
```

| 状态 | 视觉 |
|---|---|
| 默认 | 纸卡 + hairline 边 |
| hover(桌面) | 边框加深 + 快捷动作从 60% 透明升至 100% |
| selected(抽屉开) | 左缘 2px 墨色 |
| `new` | 淡入上滑 240ms |
| `revised` | 左缘金色 2px 脉冲 600ms×1,之后保留静态金缘 2s 淡出 |
| busy(AI 处理中) | 整卡 60% 透明 + 禁点 |

- 快捷动作发送的预制意图**必须带天数**:`Make Day 2 lighter…`,由按钮所在
  卡注入,避免 AI 猜错天。
- 移动端:快捷动作常驻(不靠 hover),按钮高 44px,横向可滚。

### 3.2 时段块(block)级微操作(v0.2.6 实现)

- 桌面 hover 单个时段行 → 行尾浮现两个 16px 图标:`⟳ 换一个`、`ⓘ 详情`。
- `⟳` 发送 `Replace the {time} activity on Day {n} with a different option that fits my preferences.`
- 移动端:点时段行 = 打开抽屉定位到该块(不做行内浮层)。

### 3.3 完成度进度条(TripSummary 内)

- 形态:横向 6 段(路线/酒店/餐食/交通/支付/签证),已完成段实色,未完成
  段 hairline 空心;总分数字右侧 13px。
- 配色:<40% 墨灰、40–79% 金、≥80% 苔绿;patch 后从旧值动画到新值 300ms。
- 每个未完成段可点 → 发送对应补全意图("帮我补上各天的酒店区域建议")。
- 移动端窄屏:折为 32px 环形表盘 + 缺口 chips 横滚。

### 3.4 出发准备区(Before you fly)

- 位置:Day 时间线之后、画布底部;标题行 + 复选清单(每项 = alert 或
  完成度缺口)。
- 勾选交互:勾选 → 行划线 + 苔绿 ✓ + 完成度条即时 +Δ 动画;取消勾选可逆。
  勾选状态入 `TripState`(需在 v0.2.6 给 alert 加可选 `done?: boolean`,
  向后兼容)。
- 每项右侧保留原 Tools 深链(既有 `alertActionHrefMap` 通道)。

## 第四部分:视觉系统细则(token 的"使用法",不只是值)

### 4.1 字体层级表(全站统一,R3 收编进 token)

| 层级 | 规格 | 用途 |
|---|---|---|
| Display | serif 24–28px / 650 | 页面 H1、空状态标题 |
| Headline | serif 18px / 600 | 回复块标题、Day 卡主行(15px 变体) |
| Body | sans 14px / 400 / 行高 1.55 | 正文、清单 |
| Caption | sans 12–13px / 400 / 灰 72% | 次要行、辅助说明 |
| Micro | sans 11px / 500 / 灰 60% | 模型标签、时间戳 |

### 4.2 色彩使用规则(比色值更重要)

- **朱砂**:每屏唯一主行动(nextStep 按钮/主 CTA)。禁用于大面积底色。
- **金**:变更脉冲、评分星、完成度中段。**禁止用作正文文字色**(纸底上
  对比度不足),文字一律 ink。
- **苔绿**:完成态 ✓、价格、完成度高段。
- **琥珀**:watchOut 底色(8% 透明度)与 ⚠ 图标。
- **墨色灰阶**:ink 100%(标题)/ 88%(正文)/ 72%(次要)/ 60%(micro)
  / 12%(hairline)五档,禁止随手新灰。

### 4.3 卡片与层级

- 三层纸面:page(--paper)→ card(--paper-soft + hairline)→ inset
  (卡内再嵌一层,如 watchOut 条、变更摘要卡:6% 墨色底)。
- 阴影全站禁用(与水墨纸面冲突),层级只靠底色与 hairline 表达。
- 圆角:卡 8px、sheet 12px、Pill 999px,三档之外不新增。

### 4.4 图标

- lucide 线性,仅 16/20 两档,strokeWidth 1.6;语义色跟随所在文本层级,
  不单独着色(除 ✓ 苔绿、⚠ 琥珀两个语义例外)。

## 第五部分:动效参数总表(实现即抄表)

| 场景 | 属性 | 时长/缓动 | 备注 |
|---|---|---|---|
| 卡片进入 | opacity 0→1, translateY 8→0 | 240ms ease-out | 列表 stagger 40ms |
| revised 脉冲 | 左缘 2px 金,box-shadow 扩散 | 600ms ×1 | 后静态金缘 2s 淡出 |
| 完成度变化 | width/stroke 动画 | 300ms ease-out | patch 演出的收尾 |
| 按钮 hover | background | 100ms | 按下 scale(0.98) 80ms |
| 回复分阶段 | 逐条 opacity/translateY | 60ms stagger | 总 <400ms |
| 打字指示 | 三点 8px 跳动 | 1.2s loop | 仅等待期存在 |
| 骨架 shimmer | 背景位移 | 1.2s loop | paper-soft 底 |
| Toast | 进出 translateY | 进 240 / 停 2.4s / 出 200 | 底部安全区上方 |
| Sheet(移动) | translateY | 280ms cubic-bezier(0.32,0.72,0,1) | 拖拽把手可拖 |
| reduced-motion | 全部 | 0ms | 脉冲→静态标记 |

## 第六部分:移动端交互补充

- **Chat sheet**:composer 常驻底导航上方;点击输入框 → sheet 上滑展开
  70dvh(把手可拖动收放);发送后 sheet 自动半收(30dvh),露出画布演出。
- **触达**:一切可点 ≥44×44px;Day 卡快捷动作横向滚动条隐藏但可滑。
- **键盘**:输入框聚焦时用 `visualViewport` 适配,composer 紧贴键盘顶。
- **抽屉**:保持点关闭按钮;下滑手势关闭列入 R3 之后(不在本三轮承诺)。

## 第七部分:可用性与无障碍底线

- 焦点:发送后焦点回输入框;抽屉/Sheet 开启焦点移入、Esc/关闭按钮返还。
- `aria-live="polite"` 保留在 status 行;变更摘要卡加 `aria-label="本次行程改动"`。
- 键盘可达:快捷动作、完成度分段、准备区复选全部可 Tab 聚焦。
- 对比度:执行 4.2 的"金色禁作文字"规则后,全站文本对 --paper 底 ≥4.5:1。
