# v2.6 支付闭环 — 实现计划

> 状态: P0 对齐 ✅ → P1 设计 → P2 拆分

**目标**: 酒店 booking 和服务 order 支持支付流程（create → pending → paid → refunded）
**范围**: 支付抽象层 + Mock 支付 + 状态机 + DB 记录 + 退款流程
**不做**: 真实微信/支付宝 API（需商户号）、多币种
**成功标准**: 创建支付→模拟支付成功→状态变更→退款→测试 ≥ 22

## P1 设计

### 支付状态机

```
pending_payment → paid → (refunding → refunded)
                 ↘ expired
```

### 新增模型

```python
class Payment(Base):
    id: str
    entity_type: str     # hotel_booking | service_order
    entity_id: str       # booking.id or order.id
    user_id: str
    amount: int          # in cents/分
    currency: str        # CNY
    status: str          # pending_payment|paid|refunding|refunded|expired
    provider: str        # mock|wechat|alipay
    provider_ref: str    # external payment ID
    created_at: datetime
    paid_at: datetime | None
```

### API 端点

| 方法 | 路径 | 说明 |
|------|------|------|
| POST | /payments | 创建支付（登录） |
| GET | /payments/{id} | 查询支付状态 |
| POST | /payments/{id}:pay | 模拟支付成功 |
| POST | /payments/{id}:refund | 退款 |

### Provider 架构

```
PaymentProvider (ABC)
├── MockPaymentProvider  ← 开发/测试用
├── WechatPayProvider   ← 骨架
└── AlipayProvider      ← 骨架
```
