# v2.5 酒店系统升级 — 实现计划

> 状态: P0 对齐 ✅ → P1 设计 → P2 拆分

**目标**: 酒店系统从 stub 升级为真实可用的多 provider 架构，含真实中国酒店数据种子。
**范围**: 多 provider 抽象层 + 真实数据种子（北京/上海/成都 60+ 酒店）+ 前端酒店卡片渲染
**不做**: Trip.com API 对接（需商务合作）、支付集成（v2.6）
**成功标准**: 酒店搜索返回真实酒店名/价格/图片/评分、前端展示酒店卡片、provider 可配置切换

## P1 设计

### 架构

```
HotelProvider (ABC)
├── SeedHotelProvider     ← v2.5: 真实种子数据（60+ 中国酒店）
├── StubHotelProvider     ← 保留：测试用
└── TripComProvider       ← 骨架：env 配置后启用
```

### Provider 选择逻辑

```
HOTEL_PROVIDER=tripcom  → TripComProvider (需 TRIPCOM_API_KEY)
HOTEL_PROVIDER=seed     → SeedHotelProvider (默认)
默认                    → SeedHotelProvider
```
