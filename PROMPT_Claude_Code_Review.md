## 第二步：Bug 检查清单

针对每个模块检查以下问题：

### 前端 (web/)

| 检查项 | 说明 |
|--------|------|
| SSE 流式解析 | JSON 解析是否有 try-catch？断流时是否重连？ |
| 底部导航 | mobile 底部导航与键盘弹出是否冲突？（`visualViewport` API） |
| Tab 切换 | 切换时是否清理/重置上一个 Tab 的状态？ |
| 翻译面板 | 语音识别 Web Speech API 的浏览器兼容性处理？ |
| Dashboard | 循环渲染使用 `innerHTML` 还是 DOM 操作？有无 XSS 风险？ |
| Fetch 调用 | 所有 `fetch()` 是否有 `.catch()` 或 await try-catch？ |
| 图片加载 | 图片加载失败时有无 fallback？懒加载是否正常？ |
| 输入处理 | 输入框有无防抖？中英文输入法是否触发多次事件？ |
| 缓存 Service Worker | sw.js 缓存策略是否合理？更新后是否清除旧缓存？ |

### 后端 (api/)

| 检查项 | 说明 |
|--------|------|
| 路由注册 | 所有 API 路由是否正确注册到 WSGI app？ |
| 异常处理 | API 函数有没有全局异常捕获？返回格式是否一致？ |
| 配置管理 | API keys 是否硬编码？能否从环境变量读取？ |
| 健康检查 | `/api/health` 是否返回有意义的状态信息？ |
| 翻译 API | 翻译数据读取是否有缓存？是否处理文件不存在？ |
| 地图 API | 高德 API 调用是否有超时处理？API key 是否前端暴露？ |
| 酒店/团购 API | 存根函数返回格式是否与前端期望一致？ |

### 通用

| 检查项 | 说明 |
|--------|------|
| 控制台错误 | 检查所有 `.js` 文件有无潜在运行时错误（undefined 变量、类型错误） |
| 移动端兼容 | 375px ~ 430px 宽度下是否有横向溢出？ |
| 无障碍 | 是否使用语义标签？aria 属性是否正确？ |
| 性能 | 有无不必要的重复渲染、未清理的定时器、内存泄漏？ |
